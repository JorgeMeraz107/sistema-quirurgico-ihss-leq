## 📌 Descripción General

LEQ es un sistema de escritorio desarrollado como reemplazo del antiguo sistema quirúrgico institucional construido en FoxPro hace aproximadamente 20 años por una consultora internacional.

El sistema legacy únicamente funcionaba sobre Windows XP y debía ejecutarse mediante virtualización usando VirtualBox, lo cual generaba limitaciones técnicas, problemas de mantenimiento y dependencia de infraestructura obsoleta.

La nueva versión fue desarrollada completamente en Java, permitiendo modernizar la plataforma quirúrgica institucional mediante una solución multiplataforma, escalable y mantenible.

---

# 🚀 Características Principales

- Gestión integral de operaciones quirúrgicas
- Administración completa de pacientes
- Sistema de usuarios y roles
- Permisos administrativos avanzados
- Contraseñas hasheadas y usuarios encriptados
- Dashboards estadísticos y gráficos
- Generación avanzada de reportes
- Exportación a PDF, Excel y Word
- Sistema multiplataforma
- Automatización de backups
- Integración con Data Center institucional
- Arquitectura desktop robusta
- Más de 40 interfaces funcionales

---

# 🛠️ Tecnologías Utilizadas

| Tecnología | Uso |
|---|---|
| Java SE | Desarrollo principal |
| Java Swing | Interfaces gráficas |
| JDBC | Conexión a base de datos |
| MySQL | Gestión de datos |
| MySQL Server | Gestión y administración de base de datos |
| JasperReports | Generación de reportes |
| Apache POI | Exportación Excel/Word |
| HTML | Reportes y visualización |
| Bash Scripts | Automatización y backups |

El sistema implementa múltiples mecanismos de seguridad:

- Roles y permisos
- Control de accesos

# 🗄️ Infraestructura y Automatización

El sistema se encuentra integrado con la infraestructura institucional del Data Center, incluyendo:

- Automatización de respaldos
- Backups diarios y semanales
- Scripts Bash programados
- Gestión de almacenamiento
- Operación multiplataforma
- Administración centralizada

---

# 🔒 Privacidad y Confidencialidad

Debido a políticas institucionales y protección de datos médicos, el código fuente del sistema no se encuentra publicado públicamente.

Asimismo, múltiples módulos e interfaces adicionales no pueden ser mostrados debido a la naturaleza confidencial de la información hospitalaria y de pacientes.

---

# 📈 Impacto del Proyecto

El sistema LEQ permitió:

- Modernizar un sistema quirúrgico con más de 20 años de antigüedad
- Eliminar dependencia de Windows XP
- Sustituir virtualización heredada
- Mejorar administración hospitalaria
- Optimizar generación de reportes
- Centralizar procesos quirúrgicos
- Implementar controles modernos de seguridad
